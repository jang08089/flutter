 과제를 통해 트러블 슈팅이란 단어를 접하게 되었는데 쉽게 이해한 바로는 오류 발생 -> 해결 이였다.
쉽게 우리가 코딩하면서 오류가 발생하고 오류 부분을 추적해서 그걸 해결하는 과정을 말하는 듯 하였다.
추천받은 aws s3 트러블 슈팅에 대해 찾아보았는데 스토리지 관련 문제라고 한다.

크게 발생하는 문제로는 5가지인데
1. 접근 권한 
2. 파일 업로드/다운로드 문제
3. Region 문제
4. 로그 및 모니터링 문제
5. 데이터의 손실 및 덮어쓰기

lg 의 트러블 슈팅 사례를 찾지는 못하였는데

보통 일반적 프로세스는

문제 인지 -> 원인 분석 -> 해결 조치 -> 검증 -> 재발 방지 
순으로 이루어진다고 한다.
원인 분석 단계에서는 위의 문제에 따라

 
권한 문제 → 정책 수정 (예: s3:GetObject 추가)

업로드 실패 → 멀티파트 업로드 재시도 / 네트워크 경로 점검

데이터 손실 → Versioning 복구 / Glacier 복원

리전 불일치 → 올바른 엔드포인트로 요청 변경

잘못된 설정 → 버킷 정책/Block Public Access 설정 조정

정도로 이루어진다.

주어진 aws s3가 트러블 슈팅 관련 유명사례들이 많이 있었는데 대표적인 예시로는 2017년 발생한 aws s3 장애 사건이 있다.
이로인해 netflix, 우리가 사용하는 slack 등 접속 장애가 발생하였고 이의 원인은 잘못된 명령 실행이 원인이었다.

오늘 트러블슈팅에 대해 좀 알아봤는데 단순 코드 오류 해결만 하던 나로써는 잘 와닿지 않는 느낌이었다.
